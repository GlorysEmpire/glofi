## What the app is (understanding)

**Glofi** is a **testnet MVP of an on-chain prop trading firm** on **Polygon Amoy**:

1. **Traders** pay a USDC challenge fee, get rule-bound challenge state on-chain, and (after evaluation) can become funded and claim profit splits via **GlofiPayout**.
2. **Investors** deposit Amoy USDC into **GlofiPool**, receive **GLOFI** (ERC-20), and can withdraw against free liquidity.
3. **Governance** via **ProxyGovernance** (proposals, votes, optional proxy agents for non-wallet investors).
4. **Frontend** is a Next.js App Router marketing + dapp shell (`ethers` + MetaMask): home, trader 5-step flow, investor deposit/withdraw, dashboard, governance, about, contracts-info.

**Trust model today is not “trustless end-to-end.”** Evaluation and profit submission are **`onlyEvaluator` / owner-controlled**. Contracts *record* rules and money splits; **pass/fail and PnL still depend on a privileged wallet** until the MetaTrader/oracle path exists. Marketing (“no human can override”) overstates current guarantees — fine for testnet if labeled honestly; dangerous for mainnet messaging.

**Product maturity:** solid skeleton of the full firm (5 contracts + multi-page UI + live reads), but several UI/contract **unit mismatches**, **dual local trees**, **fake dashboard governance**, and **no real automated evaluation** block “serious testnet demo” quality and all mainnet plans.

## Errors & bugs (found in code)

### E1 — Dashboard challenge NaN (N1) — confirmed root cause family

**Files:** `app/hooks/useGlofiChallenge.ts` `getTraderChallenges`, `app/dashboard/page.tsx`

- UI expects named fields and does `bn(accountSize) / 1e6`.
- **chain-firm** maps `c[0]…c[11]` from ethers `Result`.
- **propfirm** still uses `{ id, ...challengeDetails[index] }` — spreading ethers `Result` often **does not** copy named struct fields → `accountSize`/`status` undefined → **`NaN`**.
- Even with index mapping, prefer **named access** (`c.accountSize`, `c.tier`) + `ethers.formatUnits(accountSize, 6)` for USDC-scaled fields.
- If UI points at the **wrong challenge address**, IDs exist but decode fails → empty list (silent `catch`) or garbage.

**Fix direction:** single `normalizeChallenge(c, id)` using named Result props; never silent-empty without logging; pin one Challenge address.

### E2 — Trader UI shows production tiers; contracts/test fees differ

**File:** `app/trader/page.tsx` hardcodes Tier1 `$49` / `$5,000`.  
**Contract (source):** Tier1 fee `$10`, account `$100` for testing.  
**Hook fees:** tier1 `10000000` (= $10 USDC 6dec) — matches test, **not** the UI `$49`.

Success step still shows **`$5,000` funded** (matches brief “partial”). Confusing and looks like a scam if someone pays $10 expecting $5k narrative without disclaimer.

### E3 — GLOFI / USDC decimal unit mismatch (critical tokenomics)

**GlofiToken:** standard ERC-20 **18 decimals**; founding mint `15_000_000 * 10**18`.  
**GlofiPool.deposit:** mints GLOFI with **raw USDC amount (6 decimals)** → deposit of 100 USDC mints `100e6` wei ≈ **1e-10 GLOFI** under `formatEther`.  
**Frontend:** `formatEther` for balances/supply; `getUsdcDeposited` also formatted as ether though storage is USDC-scale raw; withdraw uses `parseEther`.

Effects:
- Homepage / dashboard token stats are **misleading**.
- Founding allocation **dwarfs** LP mints in unit terms (or LPs look like dust).
- Withdraw amount UX is easy to get wrong.

**This is bigger than a UI bug** — pool mint scaling should be `amount * 10**(18-6)` (or token should be 6 decimals). Needs deliberate tokenomics decision + redeploy/migration story.

### E4 — Hardcoded fake dashboard “governance”

**File:** `app/dashboard/page.tsx`  
Proposals #1/#2 with **67%/89% bars** and local `handleVote` that **never calls the chain**. Real voting is only on `/governance`. Misleading for demos/investors.

### E5 — Investment section ignores live deposits

Dashboard always shows “No investments yet” empty state even when `tokenBalance` / `usdcDeposited` are non-zero (overview cards may show numbers, investment block does not).

### E6 — Silent failures

Many `catch { return [] / return null / return '0' }` paths (challenges, deposit, withdraw, vote) with little or no user-visible reason. Matches brief **N6**.

### E7 — Network switch only on trader flow

`wallet_switchEthereumChain` to Amoy appears on trader path; investor/governance/dashboard writes can fire on wrong chain → cryptic MetaMask errors. Matches **N7**.

### E8 — Profit-split copy inconsistency in brief (product doc)

Owner brief **Business rules** once said `80/15/15`; money flow + **GlofiPayout.sol** defaults are **`80/15/5`**. Canonical = on-chain `traderSplit=80, poolSplit=15, treasurySplit=5`. Brief file on disk corrected to 80/15/5.

### E9 — Marketing vs actual evaluation

Copy: “Smart contracts evaluate your performance automatically.”  
Code: `submitEvaluation` / `submitProfit` are **`onlyEvaluator`**. Accurate messaging: contracts **enforce rules and payouts once evaluation is submitted**; evaluation itself is still privileged.

### E10 — Wallet stack minimal

MetaMask-only (`window.ethereum`), no account/chain change listeners, no WalletConnect/Rabby multi-wallet. Fine for MVP; brittle for broader testnet users.

### E11 — Governance proposal load

`/governance` loops `1..proposalCount`. If RPC fails, count can look like 0 (brief). Indexing is 1-based (`++proposalCount`) — consistent with loop. Vote errors only `console.error`.

### E12 — “Your share 0%” and “$1.00” token price

Dashboard hardcodes share `0%` and **$1.00** token value — not computed from pool.
